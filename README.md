# ReactQuest — Learn React From Zero to Internals

ReactQuest is a self-contained, interactive web-based learning application designed to teach React from absolute basics through to advanced internals (Fiber, Reconciliation, and Concurrent Rendering). It combines conceptual scaffolding via real-world analogies, interactive custom visualizations, and in-browser editable code sandboxes.

---

## 🚀 Key Features

1. **Analogy-First Explanations**: Every lesson begins with a physical analogy (e.g., spreadsheet cells for states, whiteboards for rendering, library card catalogs for Fiber) to build solid mental models before writing code.
2. **Live Code Sandbox**: An in-browser editor (CodeMirror 6) and transpiler (`@babel/standalone`) allow learners to write, run, break, and fix React components in real time with zero setup and no server-side compilation.
3. **Interactive Internals Visualizations**: Custom, bespoke visual engines explaining advanced React internals:
   * **Virtual DOM Inspector (11.1)**: Visualizes the plain JavaScript objects generated by JSX compilation.
   * **Fiber Tree walk (11.2)**: An SVG diagram animating depth-first traversal of Fiber nodes (`child`, `sibling`, `return`, `alternate` pointers) and hooks lists.
   * **Reconciliation Diff (11.3)**: Simulates the heuristic matching of DOM nodes during rendering.
   * **Concurrent Timeline (11.4)**: An "Air Traffic Control" thread timeline comparing blocking vs. concurrent time-slicing.
4. **Prerequisite-based Unlock System**: Chronological curriculum paths lock and unlock based on prerequisites. User progress is persisted locally (`localStorage`) and can be exported or reset.
5. **Calm, Focus-oriented UI**: Styled with a premium "paper-and-ink" warm palette, tailored typography (Fraunces, Inter, and JetBrains Mono), responsive columns, and strict accessibility standards (`prefers-reduced-motion`, keyboard focus navigation, screen-reader alert regions).

## 💾 User Progress & Persistence

ReactQuest has a built-in progress tracking system that runs entirely client-side in the browser:

* **Storage Mechanism**: Progress is saved via the browser's native `localStorage` API under the key `reactquest_progress`.
* **Persistence Duration**: Because it uses `localStorage`, your learning journey is stored **indefinitely** (it never expires). You can close the tab, close your browser, or restart your computer and return weeks or months later to resume exactly where you left off.
* **Incognito/Private Browsing**: If you run the application in an incognito or private window, progress will only persist for that active session and will be cleared as soon as the private browser tab/window is closed.
* **Backup & Reset**: 
  * **Export Progress**: You can export your progress as a `.json` backup file from the Curriculum Map dashboard.
  * **Reset Progress**: You can clear all completed lessons and lock states to start the quest fresh by clicking the **Reset Progress** button.

## 🛠️ Technology Stack

* **Core**: React 18, React DOM, React Router v6
* **Transpilation**: `@babel/standalone` (compiled in-browser)
* **Editor**: CodeMirror 6 (`@uiw/react-codemirror` and `@codemirror/lang-javascript`)
* **Styling**: Vanilla CSS (CSS custom variables token system)
* **Text**: `react-markdown` (for rendering lesson texts)
* **Bundler**: Vite

## 💻 Getting Started

### Prerequisites

* [Node.js](https://nodejs.org/) (v18 or higher recommended)

### Installation

1. Clone or navigate to the project directory:
   ```bash
   cd ReactLearner
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

### Development Commands

* **Start the development server**:
  ```bash
  npm run dev
  ```
  Open `http://localhost:5173` in your browser.

* **Run a production build**:
  ```bash
  npm run build
  ```

* **Locally preview the production build**:
  ```bash
  npm run preview
  ```

## 🐳 Containerization & CI/CD

ReactQuest supports local containerized development and automated integration tests:

### Docker Development Environment
Start the application locally using Docker Compose. Hot Module Replacement (HMR) is pre-configured to work reliably over host-to-container mounts (polling file-watching is active for Windows host compatibility).

1. **Start all services** (boots the React frontend on `http://localhost:5173` and the mock stub backend API on `http://localhost:8080`):
   ```bash
   docker compose up
   ```
2. **Rebuild container images**:
   ```bash
   docker compose build
   ```
3. **Run developer tools**:
   To programmatically check curriculum prerequisite paths inside a container on demand, execute the validator service:
   ```bash
   docker compose --profile tools run validator
   ```

### GitHub Actions CI
Every commit push and pull request triggers the automated [CI/CD Build Workflow](file:///.github/workflows/ci.yml) which performs:
1. **Node Build Validation**: Runs dependency installation (`npm ci`), checks for the presence of a lint script, compiles the production bundle (`npm run build`), and saves the output directory as a workflow artifact.
2. **Docker Build Verification**: Runs `docker compose build` inside a hosted runner to ensure all configuration and image files build successfully.
